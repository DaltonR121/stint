stint

I forgot to start my timer for the 100th time. So I built one that doesn't need starting.

Stint is an open-source, local-first time tracker built in Rust. Its killer feature: automatic time tracking via shell hooks. Open a terminal in a project directory and the clock starts. Switch projects — it switches too. Close the last terminal — it stops.

Install

One-liner for Linux & macOS — detects your OS, sets up the apt repository on Debian/Ubuntu:

$ curl -fsSL https://daltonr121.github.io/stint/install.sh | sudo sh

Prefer to inspect first?

$ curl -fsSL https://daltonr121.github.io/stint/install.sh -o install.sh
$ less install.sh
$ sudo sh install.sh

Other methods:

# Rust developers
$ cargo install stint-cli

# Debian/Ubuntu — apt repo directly (this site hosts it)
$ curl -fsSL https://daltonr121.github.io/stint/stint.gpg | sudo tee /usr/share/keyrings/stint.gpg > /dev/null
$ echo "deb [signed-by=/usr/share/keyrings/stint.gpg] https://daltonr121.github.io/stint stable main" | sudo tee /etc/apt/sources.list.d/stint.list
$ sudo apt update && sudo apt install stint

Windows and macOS binaries are on GitHub Releases.

Quick start

# One-time setup — hooks into your shell prompt
$ stint init bash    # or: stint init zsh / stint init fish

# Then just work. Tracking is automatic.
$ stint summary
$ stint dashboard

Why Stint?

  • Zero-friction tracking — hooks into your shell, no timers to remember
  • Multi-project — each terminal tracks its own project independently
  • Local-first — your data lives in SQLite on your machine, no telemetry
  • Fast — the shell hook executes in under 2ms
  • TUI dashboard, CSV reports, and a VS Code extension
